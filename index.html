01 Electronics Fundamentals DEFINITELY ASKED ▶
Ohm's Law & Power
V = IR  |  P = VI  |  P = I²R  |  P = V²/R
  • V = Voltage (Volts) — the push. I = Current (Amps) — the flow. R = Resistance (Ohms) — the friction
  • 5V across 100Ω → I = 5/100 = 50mA
  • 2A through 10Ω → P = 4×10 = 40W (heat)
  • KCL — current in = current out at any node
  • KVL — sum of all voltages around a loop = 0
Passive Components
  • Resistor — limits current. Series: R1+R2. Parallel: (R1×R2)/(R1+R2)
  • Capacitor — stores charge. DC steady state = open circuit. At high freq = short circuit
  • Electrolytic caps are polarised — reversed = overheats, swells, fails
  • 100nF ceramic decoupling cap — goes RIGHT NEXT to IC VCC pin, filters switching noise
  • Inductor — opposes current change. DC = short. High freq = open
  • Diode — one direction only. Silicon forward voltage = 0.6–0.7V. Cathode = stripe
Transistors
  • BJT NPN — current-controlled. Base drives Collector→Emitter. VBE ≥ 0.7V to turn on
  • MOSFET N-channel — voltage-controlled. Gate HIGH → Drain to Source conducts. Almost no gate current
  • MOSFETs used in power circuits — lower loss, faster switching than BJT
Power Supply
  • LDO — simple, burns excess as heat. Efficiency = Vout/Vin. Heat = (Vin−Vout) × I
  • 5V→3.3V at 500mA: heat = 1.7×0.5 = 0.85W, efficiency = 66%
  • Buck converter — switching, steps DOWN efficiently (85–95%). Vout = Vin × duty cycle
  • Boost converter — stepping UP (battery→USB)
  • Board dead → check power rails first. Always. Every time.
02 Lab Instruments DEFINITELY ASKED ▶
DMM Modes
  • DC Voltage (V—) — measure rails, logic levels. Red to +, black to GND
  • Resistance (Ω) — POWER OFF first. Measure component values, trace continuity
  • Continuity (beep) — beeps if <30Ω. Check solder joints, traces, shorts
  • Diode test — silicon shows 0.6–0.7V forward. OL in reverse = good diode
  • Current (A) — break the circuit, insert DMM IN SERIES. Use correct port
Oscilloscope
  • X axis = time/div (timebase). Y axis = V/div (voltage scale)
  • Trigger stabilises waveform — set to ~50% of signal amplitude, rising edge
  • Always connect probe ground clip to circuit GND first
  • DMM shows average — scope shows transients, noise, spikes, ripple
  • Use scope for: power rail noise, clock quality, SPI/I2C/UART signals, glitches, PWM

  • Shows frequency domain — X axis = frequency, Y axis = amplitude (dBm)
  • Each peak = a frequency present in the signal

03 Hardware Debugging Methodology DEFINITELY ASKED ▶
The Process — say this in order
  • 1. Define the symptom precisely — what exactly fails, under what conditions, how reproducibly
  • 2. Check power first — always — every rail, every pin, ground continuity
  • 3. Visual inspection — burnt marks, swollen caps, solder bridges, cold joints, wrong orientation
  • 4. Reproduce the failure — cannot root-cause what you can't reproduce consistently
  • 5. Isolate the domain — power? comms? firmware? mechanical?
  • 6. Divide and conquer — split circuit in half, test midpoint, eliminate half the possibilities
  • 7. Measure — never assume — probe it. Every hypothesis tested with instrument
  • 8. Document everything — every measurement, every result
Common Failure Modes
  • Cold solder joint — dull, grey, grainy (not shiny). High resistance. Intermittent. Reflow to fix
  • Solder bridge — blob connecting adjacent pins/pads. Short circuit. Remove with wick
  • Open trace — broken PCB track. Continuity test shows open. Fix with bodge wire
  • ESD damage — IC appears fine but dead. No visual sign. Requires IC replacement
  • Reversed polarity — cap or diode backwards. Overheating, bulging, failure
  • Wrong component — same package, wrong value. Measure actual value vs BOM
  • Lifted pad — pad separated from PCB. Looks soldered but no connection
Protocols — Debugging Each
  • UART — 2 wires TX/RX, no clock, async. Both ends must match baud rate. Idle = HIGH. Check: baud match, voltage levels, TX↔RX not swapped
  • I2C — 2 wires SDA/SCL. Open-drain — needs pull-up resistors (4.7kΩ). Multi-device via 7-bit address. Check: pull-ups present? correct address? ACK returned?
  • SPI — 4 wires MOSI/MISO/SCK/CS. Fast, full-duplex. CS goes LOW to select. Check: CS toggling? correct CPOL/CPHA mode? frequency within device max?
  • I2C bus lockup — SDA stuck LOW. Fix: clock SCL 9 times then send STOP
04 Failure Analysis (FA) DEFINITELY ASKED — YOU FORGOT LAST TIME ▶
You said you forgot FA points in round 1. Read this section carefully and say it out loud once before the interview.
FA Flow — 6 Steps (memorise this order)
  • 1. Visual inspection — naked eye → magnifier → microscope. Document with photos before touching anything
  • 2. Electrical characterisation — DMM + scope. Reproduce the failure. Measure all rails vs expected
  • 3. Fault isolation — narrow down to component, joint, or circuit area using a good unit as reference
  • 4. Physical analysis — X-ray (non-destructive), cross-section (destructive), SEM/EDS if needed
  • 5. Root cause identification — classify: design defect / material defect / process defect / ESD / workmanship
  • 6. Corrective action — 8D report, FMEA update, ECN if design needs to change
FA Tools
  • X-ray — non-destructive. Views solder under BGA/QFN without removing component. Detects voids, bridges, missing balls
  • Cross-section — destructive. Cut, polish, view under microscope. Shows internal solder joint structure, cracks, voids
  • SEM — Scanning Electron Microscope. Very high magnification (up to 100,000×). Shows cracks, fractures, surface morphology
  • EDS — used WITH SEM. Identifies chemical elements present. Confirms contamination, wrong alloy, corrosion products
What an FA Report Contains
  • Unit ID, build date, batch, which test failed
  • Exact symptom and conditions
  • All measurements taken with photos/data
  • Fault isolation steps
  • Root cause statement with supporting evidence
  • Root cause classification (design/material/process/ESD)
  • Corrective action recommendation
Key Terms
NPI = New Product Introduction FA = Failure Analysis ICT = In-Circuit Test (before assembly) FCT = Functional Test (after assembly) AOI = Automated Optical Inspection ECN = Engineering Change Notice BOM = Bill of Materials PCBA = PCB Assembly Yield = Pass% of total tested Pareto = 80% failures from 20% causes
05 8D Problem Solving & FMEA DEFINITELY ASKED ▶
8D — All 8 Steps
  • D1 — Form cross-functional team
  • D2 — Describe problem with data (5W2H: Who/What/Where/When/Why/How/How many)
  • D3 — Interim Containment Action (ICA) — TEMPORARY fix. Quarantine suspect units, add 100% inspection
  • D4 — Root cause — use 5-Why, Fishbone/Ishikawa diagram
  • D5 — Choose Permanent Corrective Action (PCA). Verify it works
  • D6 — Implement PCA. Update work instructions, BOMs, assembly drawings
  • D7 — Prevent recurrence — update FMEA, control plans, share across product lines
  • D8 — Close report, recognise team
What's the difference between containment and corrective action?
Containment (D3) is immediate and temporary — stops the bleeding right now. Example: quarantine the batch, add manual inspection. Corrective action (D5/D6) is the permanent fix that eliminates the root cause so this failure never happens again.
FMEA
RPN = Severity × Occurrence × Detection (max = 1000)
  • Severity (S) — how bad is the effect? 1–10. 9–10 = safety/regulatory failure
  • Occurrence (O) — how often does it happen? 1–10. High = frequent
  • Detection (D) — how likely to be caught? 1 = almost certain to catch. 10 = impossible to detect
  • Higher RPN = higher priority. Reduce RPN by improving detection or reducing occurrence
  • FMEA is proactive — done before failures happen to predict and prevent them
5-Why Example
  • Board dead → LDO output 0V → input cap shorted → reversed polarity → no marking in assembly drawing → silkscreen lacked "+" symbol
  • Root cause: missing polarity marking in PCB design → update silkscreen guideline and design checklist
06 Sensors — ALS & Proximity LIKELY ASKED ▶
Ambient Light Sensor (ALS)
  • Measures light intensity in lux. Used in phones to auto-adjust screen brightness
  • Photodiode generates current proportional to light → ADC converts to lux value
  • Interface: I2C. Spectral response matches human eye (~550nm green)
  • Integration time = how long ADC collects charge. Longer = more sensitive in dim light but slower
ALS FA Issues
  • Protective film over sensor → reads 0 lux. Remove film and retest
  • Wrong I2C address (address pin pulled wrong way)
  • Sensor not initialised in firmware — still in standby, returns stale 0
  • Integration time not configured → saturated or too slow
  • Physical damage or contamination on optical window
Proximity Sensor
  • Turns off screen when phone held to ear. Uses IR LED + IR photodetector
  • IR LED pulses → reflected IR measured → more reflection = object closer
  • Crosses near/far threshold → generates interrupt → firmware turns off screen
  • Often combined with ALS in same IC (APDS-9930, LTR-559)
Proximity FA Issues
  • Crosstalk — IR reflects off device housing, triggers without real object. Most common NPI issue
  • IR LED not driving — wrong current resistor or open circuit. Check voltage across LED on scope
  • Threshold registers wrong — triggers too early or not at all
  • Black tape/adhesive over sensor window from assembly
  • Ambient sunlight saturating photodetector → false near reading
07 Analog & Digital Circuits DEFINITELY ASKED ▶
Analog vs Digital
  • Analog — continuous range of values. Microphone, temperature sensor output, audio
  • Digital — only HIGH or LOW. UART TX, GPIO, SPI clock
  • ADC — converts analog → digital. 12-bit = 4096 levels. Resolution = Vref/2ⁿ
  • 12-bit ADC, 3.3V ref: 3.3/4096 = 0.8mV per step
  • DAC — converts digital → analog. Used in audio output, signal generation
Logic Voltage Levels
  • 5V TTL — HIGH >2V, LOW <0.8V
  • 3.3V CMOS — HIGH >2V, LOW <0.8V. Max input = 3.3V (unless 5V-tolerant)
  • 5V signal into 3.3V GPIO → can permanently damage the pin
  • Fix: voltage divider, level shifter IC (TXS0102), or Zener clamp
  • Some STM32 pins marked "FT" = 5V tolerant. Check datasheet per pin
Logic Gates
  • AND — all inputs HIGH → output HIGH
  • OR — any input HIGH → output HIGH
  • NAND — AND + NOT. Universal gate (can build anything from NAND)
  • XOR — output HIGH when inputs DIFFERENT. Used in error detection
PWM
  • Digital signal switching HIGH/LOW rapidly. Duty cycle = % time HIGH
  • 50% duty cycle at 3.3V → average 1.65V to load
  • Used for: LED dimming, motor speed control, servo position, DAC approximation

  • Communicates over Modbus RTU — industrial serial protocol over UART/RS485

FreeRTOS — what is it and how did you use it?
  • Real-time OS for microcontrollers. Multiple tasks run concurrently managed by a scheduler
  • Key primitives: tasks (like threads), queues (pass data between tasks), semaphores (synchronisation), task priorities
  • Used on ESP32 and nRF52 — sensor polling in one task, BLE communication in another, so neither blocks the other
  • Shared data between tasks must be protected with mutex to avoid race conditions
I2C vs SPI — when to choose each?
  • I2C — 2 wires, multi-device via addressing, needs pull-ups, max 400kHz standard. Choose for: slow sensors, many devices, limited pins
  • SPI — 4 wires, faster (tens of MHz), full-duplex, no pull-ups. Choose for: displays, flash memory, fast ADCs

09 Real-World Scenario Questions EXPECT THESE IN DEEP TECHNICAL ▶
Board is completely dead. Walk me through your debug process.
  • Measure input voltage at connector — correct voltage arriving?
  • Measure LDO/buck output — rail at correct voltage?
  • Measure VCC at main IC pin — not just at regulator, check it arrives at IC
  • Check ground continuity
  • If rails correct → check enable pins, reset line, communication lines
  • Visual inspection — burnt components, reversed cap, solder bridge
Two identical boards — one works, one doesn't. How do you approach?
  • Design is probably fine — failure is likely assembly defect or component variation
  • Use working board as golden reference
  • Measure same test points on both — first divergence = fault location
  • Check: power rails → communication signals → individual IC pins
Board works at room temp, fails after 30 minutes. What do you suspect?
  • Thermal failure — component marginal at room temp, fails when hot
  • Use thermal camera to find hot components
  • Suspects: electrolytic cap (capacitance drops with heat), LDO going into thermal shutdown, solder joint opening with thermal expansion
  • Calculate: at your load current, is LDO power dissipation within its thermal rating?
30% of boards fail ICT resistance check on R15. X-ray shows no solder issues. What next?
  • 30% is systematic — not random. Focus on the cause that affects ALL failing boards
  • Measure actual resistance of R15 on failed boards with DMM — wrong value = wrong component reel loaded
  • Check ICT fixture probe for R15 — worn/bent probe causes false fails
  • Check schematic — is anything in parallel with R15 that affects the ICT measurement?
Phone screen turns off randomly during calls when held to ear.
  • Proximity sensor issue — triggering when it shouldn't
  • Check: threshold registers too sensitive?
  • Check: IR crosstalk from device housing reflecting back?
  • Check: protective film or adhesive over sensor window from assembly?
  • Test in open air dark room — if sensor reads "near" with nothing nearby → crosstalk
  • Test multiple units — one unit = physical defect. All units = firmware threshold or design issue
You replace a cap and the board works. Are you done?
No. That fixes the symptom, not the root cause. Ask: why did the cap fail? Wrong voltage rating? Reversed polarity? Counterfeit? Thermal overstress? The answer determines risk to other boards in the batch. Document, check other boards, initiate corrective action.
Twisted Questions — Quick Answers
Fully charged capacitor in DC — current flowing?
Zero. Cap = open circuit in DC steady state. Current only flows while voltage is changing.
DMM shows 3.3V on GPIO — does that confirm it's driving correctly?
Not necessarily. DMM has 10MΩ impedance — barely loads the pin. Voltage might collapse under real load. Measure with actual load connected or use scope.
Continuity beep between VCC and GND — always a short?
Not always. Could be legitimate: LED, small resistor, protection diode. Switch to resistance mode, read the actual value. True short = near 0Ω + component getting hot immediately.
LED: 2V forward, 20mA, powered from 5V. Series resistor?
V across R = 5−2 = 3V. R = 3/0.02 = 150Ω. Power in R = 3×0.02 = 0.06W. Standard ¼W resistor is fine.
LDO: 5V in, 3.3V out, 500mA. How much heat?
Heat = (5−3.3) × 0.5 = 0.85W. Efficiency = 3.3/5 = 66%.
FMEA: S=8, O=4, D=3. RPN?
RPN = 8×4×3 = 96. Moderate. But Severity=8 means if it occurs, it's serious — review even at moderate RPN.
200 boards, 174 passed. Yield?
174/200 × 100 = 87%. Failed = 26. Analyse all 26 for Pareto.
